# Welcome to NexeCare

NexeCare is a comprehensive healthcare management SaaS platform designed specifically for parents, caregivers, and medical professionals caring for children with multiple disabilities. Born from the collaboration between VelTech and Fundación Nexe, NexeCare combines technology and social commitment to provide personalized, high-quality care management.

<Note>
  NexeCare simplifies daily life for those who need it most by providing intuitive tools to manage medication schedules, track intake, and coordinate care across multiple caregivers.
</Note>

## Who is NexeCare For?

NexeCare is designed for:

- **Parents and Guardians**: Manage your child's medication schedules with confidence and reduce errors
- **Caregivers**: Stay synchronized with the care team and track medication intake in real-time
- **Medical Professionals**: Monitor treatment adherence and collaborate with families effectively
- **Care Coordinators**: Oversee multiple children and ensure comprehensive care management

## Architecture Overview

NexeCare is built with modern web technologies to provide a fast, reliable, and scalable platform:

### Frontend Stack

- **React 19** with **Vite** for fast development and optimal performance
- **Tailwind CSS** for responsive, mobile-first design
- **FullCalendar** for interactive calendar views
- **Supabase Auth Helpers** for seamless authentication
- **React Router** for smooth navigation

### Backend Stack

- **Spring Boot** (Java) for robust REST API services
- **JPA/Hibernate** for database operations
- **RESTful architecture** with dedicated controllers for medications, treatments, and intake tracking

### Authentication & Integration

- **Supabase** for secure authentication and user management
- **Google OAuth 2.0** for single sign-on
- **Google Calendar API** for event synchronization with calendar scopes

### Database Models

The platform uses a relational database with key entities:

```java
// Child Profile
public class Child {
    private int id;

// Medication Details
public class Medication {
    private int id;

    private String doseUnit;
    private String type;
    private String frequency;
    private int minStock;
    private int stock;
    private Date createdAt;
    private Date updatedAt;

// Treatment Plan
public class Treatment {
    private int id;
    private int childId;
    private int medicationId;
    private Date startDate;
    private Date endDate;
    private int specificDose;
    private String instructions;
}
```
